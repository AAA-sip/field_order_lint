# Документация параметров для field_order_lints

Эти параметры добавляются в файл `analysis_options.yaml` в секцию `custom_lint: rules:`.

## 1. field_order_by_type
Сортирует поля в классе по модификаторам, типам и алфавиту.

Параметры:
- `type_priority` (List): Список типов или групп типов в порядке их приоритета.
- `modifier_priority` (List): Порядок групп модификаторов. Доступные значения: `static`, `final`, `mutable`, `late`.

Пример:
  rules:
    - field_order_by_type:
        type_priority:
          - [int, double, float, num] # Группа 0 (самый высокий приоритет)
          - String                    # Группа 1
          - DateTime                  # Группа 2
        modifier_priority:
          - static
          - final
          - mutable
          - late

## 2. parameter_order_by_type
Сортирует именованные параметры в функциях и методах.

Параметры:
Использует те же `type_priority`, что и правило выше. Можно настроить отдельно:

  rules:
    - parameter_order_by_type:
        type_priority:
          - String
          - int

## 3. constructor_order_by_field
Проверяет, чтобы порядок `this.field` в конструкторе совпадал с порядком полей в классе.
Параметров нет.

  rules:
    - constructor_order_by_field

## 4. field_separator_between_groups
Требует пустую строку между разными группами модификаторов (например, между static и final).
Параметров нет.

  rules:
    - field_separator_between_groups

---
### Совет по синтаксису YAML:
Если вы не используете параметры, пишите просто список:
  rules:
    - field_order_by_type
    - parameter_order_by_type

Если параметры нужны, используйте формат словаря:
  rules:
    - field_order_by_type:
        type_priority: [int, String]
